# PRODUCT_CONTRACT

## Назначение документа
Этот документ фиксирует базовый контракт сущности `Product` для проекта сайта по продаже и выкупу б/у техники Apple.

Главная цель документа:
- у товара должен быть единый контракт во всем проекте
- storefront, admin и core не должны использовать разные версии одной и той же сущности
- любое изменение структуры товара считается критичным изменением
- изменения по `Product` нельзя делать случайно и локально без оценки последствий

---

## Роль Product в проекте
`Product` — это центральная сущность каталога.

Через `Product` работают:
- каталог товаров
- карточка товара
- admin-список товаров
- создание товара
- редактирование товара
- скрытие / показ товара
- работа с изображениями
- фильтрация и сортировка
- отображение цены и состояния
- связанные товары и товарные подборки

Если контракт `Product` становится нестабильным, начинает ломаться сразу несколько зон проекта.

---

## Главный принцип
У `Product` должен быть **один источник истины**.

Нельзя допускать ситуацию, когда:
- в storefront один набор полей
- в admin другой набор полей
- в локальном компоненте появляется “временная” версия товара
- часть полей понимается по-разному в разных частях проекта

## Предварительное описание полей

### 1. id
Назначение:
уникальный идентификатор товара внутри проекта.

Правила:
- `id` должен быть **строкой**
- нельзя смешивать `string` и `number`
- если в каком-то месте системы приходит числовой id, это считается проблемой data flow
- все зоны проекта должны работать с единым строковым форматом

Пример:
- `"iphone-13-001"`
- `"mbp-2019-16-003"`

---

### 2. slug
Назначение:
человекочитаемый идентификатор для URL карточки товара.

Правила:
- `slug` используется в адресе страницы товара
- `slug` должен быть стабильным
- `slug` не должен зависеть от случайных изменений UI
- логика генерации `slug` должна быть общей
- нельзя в одной зоне проекта генерировать `slug` так, а в другой иначе

Пример:
- `iphone-13-pro-128-graphite`
- `macbook-pro-16-2019-i7-16-512`

---

### 3. name
Назначение:
основное отображаемое название товара.

Правила:
- поле должно быть понятным человеку
- название должно подходить и для каталога, и для карточки товара, и для admin
- `name` не должно быть слишком технически сырым
- не стоит перегружать поле случайным набором характеристик без логики

---

### 4. category
Назначение:
тип товара внутри каталога.

Примеры категорий:
- `iphone`
- `ipad`
- `macbook`
- `imac`
- `apple-watch`
- `airpods`
- `accessories`

Правила:
- категории должны быть ограниченным и понятным набором значений
- нельзя допускать хаоса вроде смешения “MacBook”, “macbook”, “ноутбуки Apple” и т.д.
- логика категорий должна быть общей для storefront и admin

---

### 5. brand
Назначение:
бренд товара.

Для текущего проекта базово это:
- `Apple`

Правила:
- поле можно хранить явно, даже если проект сфокусирован на Apple
- это дает более чистую модель данных и упрощает возможное расширение логики в будущем

---

### 6. model
Назначение:
модель устройства без полного маркетингового названия.

Примеры:
- `iPhone 13 Pro`
- `MacBook Pro 16`
- `iPad Air 5`

Правила:
- `model` не должен заменять `name`
- `model` нужен как отдельное поле для удобства фильтрации, отображения и структуры данных

---

### 7. price
Назначение:
актуальная цена товара.

Правила:
- цена должна храниться как число
- форматирование в рубли должно происходить на уровне отображения, а не в самих данных
- нельзя хранить цену строкой вроде `"89 990 ₽"`

Правильно:
- `89990`

Неправильно:
- `"89 990"`
- `"89 990 ₽"`

---

### 8. oldPrice
Назначение:
старая цена до скидки или уценки.

Правила:
- поле необязательное
- должно храниться как число
- если старой цены нет, поле лучше не заполнять

---

### 9. images
Назначение:
массив изображений товара.

Правила:
- `images` должен быть массивом
- даже если у товара одна фотография, логика все равно должна быть массивом
- нельзя делать в одной части проекта `image`, а в другой `images`
- порядок изображений должен быть предсказуемым
- первое изображение обычно считается главным

Пример:
```ts
images: [
  "/uploads/iphone-13-1.jpg",
  "/uploads/iphone-13-2.jpg"
