# ROADMAP

## Этап 1. Подготовка проекта
Цель: собрать правильный фундамент до начала активной разработки.

Что входит:
- настройка компьютера и среды разработки
- создание нового репозитория
- запуск нового Next.js-проекта
- создание базовой структуры папок
- создание документов проекта
- фиксация зон ответственности чатов
- подготовка главного промта

## Этап 2. Архитектурный каркас
Цель: определить базовую логику проекта до реализации страниц.

Что входит:
- карта страниц сайта
- список основных сущностей
- определение структуры Product
- определение структуры каталога
- определение базового data flow
- разделение public / admin / core зон

## Этап 3. MVP публичной части
Цель: собрать первую рабочую версию сайта.

Что входит:
- главная страница
- каталог товаров
- карточка товара
- страницы delivery / payment / contacts
- базовая структура layout
- header и footer

## Этап 4. MVP admin-зоны
Цель: сделать удобное локальное управление товарами.

Что входит:
- список товаров
- создание товара
- редактирование товара
- удаление товара
- скрыть / показать товар
- базовая работа с изображениями

## Этап 5. Выкуп / trade-in
Цель: реализовать отдельный пользовательский сценарий выкупа техники.

Что входит:
- страница trade-in
- форма заявки
- описание процесса
- коммерческие блоки доверия
- подготовка к расширению логики заявок

## Этап 6. Улучшение UX и структуры
Цель: сделать сайт сильнее конкурентов по удобству и качеству.

Что входит:
- улучшение структуры каталога
- фильтры и навигация
- улучшение карточки товара
- улучшение визуального стиля
- адаптация под мобильные устройства
- усиление доверительных блоков

## Этап 7. SEO и контент
Цель: подготовить сайт к поисковому трафику и коммерческой подаче.

Что входит:
- SEO-структура
- meta title / description
- контентные страницы
- FAQ
- коммерческие тексты
- усиление московского фокуса при работе по всей России

## Этап 8. Техническое усиление
Цель: подготовить проект к стабильной работе и дальнейшему росту.

Что входит:
- рефакторинг по мере необходимости
- стабилизация data flow
- подключение Codex в роли исполнителя
- подготовка к деплою
- публикация проекта
- дальнейшее масштабирование

## Главный принцип
Сначала структура и архитектура, потом активная разработка.  
Сначала MVP, потом расширение.  
Любые критичные изменения сначала проходят через главный координационный чат.
